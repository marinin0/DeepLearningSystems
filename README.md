# demucs
Demucs предоставляет модель для разделения музыкальных треков на отдельные инструменты (ударные, бас, вокал и другое).
# Предварительные требования
1) Docker Desktop: Убедитесь, что он установлен и запущен. (Скачать: https://www.docker.com/products/docker-desktop/). При установке или первом запуске Docker предложит: **"Use the WSL 2 based engine"**. Нужно включить интеграцию с WSL 2. Это обеспечит правильность работы нейросети.
2) Git для операций с репозиторием. (Скачать: https://git-scm.com/download/win)
# Запуск проекта
1) Открыть терминал (cmd).
2) Перейти в нужный диск или папку:
```
   cd C:\
   mkdir dev
   cd dev
```
3) Склонировать данный репозиторий
```
git clone https://github.com/marinin0/DeepLearningSystems.git
```
4) Перейти в папку
```
cd DeepLearningSystems
```
5) Собрать Docker-образ.
```
docker build -t audio-project .
```
6) Запустить обработку и тесты.
```
docker run --rm audio-project
```

# Устранение неполадок
### Отсутствие модели
В данном репозитории веса модели уже включены в папку `models/`. Однако, если при сборке Docker возникает ошибка или папка оказалась пустой, выполните следующие шаги:

1. **Скачайте модель вручную**:
   Используйте официальную прямую ссылку на Hybrid Transformer Demucs (HT Demucs) (нужен ВПН):
   [Скачать модель (955717e8-8726e21a.th)](https://dl.fbaipublicfiles.com/demucs/hybrid_transformer/955717e8-8726e21a.th)

2. **Разместите файл**:
   Создайте в корне проекта папку `models` (если её нет) и положите скачанный файл туда. Путь должен выглядеть так:
   `DeepLearningSystems/models/955717e8-8726e21a.th`

3. **Пересоберите образ**:
   ```bash
   docker build -t audio-project .
### Ошибка "Docker daemon is not running"
Убедитесь, что приложение **Docker Desktop** запущено. Иконка кита в трее должна быть статичной.

### Настройка ресурсов (RAM)
Для работы нейросети требуется около **4 ГБ свободной оперативной памяти**. Если контейнер вылетает на этапе "Separating", попробуйте закрыть браузер или другие тяжелые программы. В WSL 2 память выделяется динамически.
